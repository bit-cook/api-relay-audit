# v2.3: Local 14-step AI API relay security audit

API Relay Audit v2.3 is a local security audit tool for AI API relays and LLM
proxies. It detects prompt injection, model substitution, tool-call rewriting,
SSE anomalies, error leakage, upstream channel mismatch, latency variance, and
Web3 wallet risks. Your API key is sent only to the relay URL you choose.

## Quick Start

```bash
curl -sO https://raw.githubusercontent.com/toby-bridges/api-relay-audit/master/audit.py
python audit.py --key <YOUR_KEY> --url <BASE_URL> --output report.md

# Web3 / wallet users
python audit.py --key <YOUR_KEY> --url <BASE_URL> --profile web3 --output report.md
```

## What Changed

- Local 14-step audit coverage for AI API relays and LLM proxies.
- Prompt injection, prompt extraction, instruction conflict, and jailbreak
  probes.
- Model identity and upstream channel mismatch checks.
- Tool-call rewriting detection for package-command tampering.
- Error-response leakage checks for credential, header, stack-trace, path, and
  internal-field exposure.
- Anthropic-style SSE stream integrity checks.
- Web3 wallet-safety probes with `--profile web3` and `--profile full`.
- OpenClaw and Hermes Agent skill distribution documentation.
- GitHub Pages SEO/GEO surfaces: English homepage, Chinese page, guide pages,
  sitemap, canonical links, and structured data.
- Community-health docs and non-code contributor entry points.

## Important Limits

- API Relay Audit does not certify that a relay is safe.
- It does not replace manual security review or operational monitoring.
- `inconclusive` is not `clean`; blocked probes and ambiguous responses stay
  visible in the report.
- Do not publish API keys, wallet material, private relay URLs, or raw traffic
  captures in issues or examples.
